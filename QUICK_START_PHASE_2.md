## Understanding Phase 2 Improvements

### Before Phase 2 (Static)
- Questions: Hardcoded "Question 1, 2, 3"
- Recommendations: Template-based
- Routing: Fixed logic

### After Phase 2 (Intelligent) ✨
- Questions: **Gemini generates adaptive follow-ups** based on your answers
- Recommendations: **Gemini synthesizes all responses** into personalized architecture
- Routing: **Gemini intelligently decides** which agents handle which tasks

## Gemini Models Used

We use **Google Gemini 2.0 Flash**:
- Fast (~50-100ms per request)
- Cost-effective
- Optimized for multi-turn conversations
- Available in `orchestration/llm.py` (line: `MODEL_NAME = "gemini-2.0-flash"`)

To use a different model:
```python
# In orchestration/llm.py
MODEL_NAME = "gemini-1.5-pro"  # More powerful, slower, higher cost
```

## Troubleshooting Phase 2

### "APIConnectionError: Failed to connect to Gemini"
- Check `GEMINI_API_KEY` in `orchestration/.env`
- Verify internet connectivity
- The system will gracefully fall back to default questions

### "KeyError: 'content'"
- Ensure you're sending `interview_id` in the request body
- Example: `{"interview_id": "...", "answer": "your answer"}`

### "HTTP 422 Unprocessable Entity"
- Check your JSON format is valid
- Use `-H "Content-Type: application/json"`

### Server not starting?
```bash
# Check Python syntax
source venv/bin/activate
python -m py_compile orchestration/llm.py orchestration/agents.py orchestration/interview.py

# Check dependencies
pip list | grep google-generativeai

# View detailed error
python orchestration/server.py --verbose
```

## Example Complete Interview (5-7 minutes)

```bash
# Terminal 3: Run this script
#!/bin/bash

INTERVIEW_ID=""

# Start
echo "Starting interview..."
START=$(curl -s -X POST http://localhost:8001/interview/start \
  -H "Content-Type: application/json" \
  -d '{"user_id":"demo"}')
INTERVIEW_ID=$(echo $START | grep -o '"interview_id":"[^"]*' | cut -d'"' -f4)

# Q1
echo "Q1: $(echo $START | grep -o '"question":"[^"]*' | cut -d'"' -f4)"
read A1
curl -s -X POST http://localhost:8001/interview/$INTERVIEW_ID/answer \
  -H "Content-Type: application/json" \
  -d "{\"interview_id\":\"$INTERVIEW_ID\",\"answer\":\"$A1\"}" > /tmp/q2.json

# Q2 (Gemini generated!)
echo "Q2: $(cat /tmp/q2.json | grep -o '"question":"[^"]*' | cut -d'"' -f4)"
read A2
curl -s -X POST http://localhost:8001/interview/$INTERVIEW_ID/answer \
  -H "Content-Type: application/json" \
  -d "{\"interview_id\":\"$INTERVIEW_ID\",\"answer\":\"$A2\"}" > /tmp/q3.json

# ... continue pattern for Q3-Q7
# Final response will have: "status":"complete", "recommendation":"..."
```
